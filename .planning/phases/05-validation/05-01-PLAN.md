  <action>
Create `models/validate.py` with the following structure:

1. **Imports and sys.path setup**: Follow the same pattern as `run.py` for path setup. Import from `models.calibrate` (_fit_calibration, _get_anchor_raw_score, DEFAULT_ANCHORS, calibrate, get_calibration_coefficients, compute_bootstrap_ci, score_to_zone), from `models.ensemble` (compute_ensemble, _build_raw_history, _run_models_on_slice, _rename_columns_for_models, _ensure_models_registered), from `models.config` (MODEL_WEIGHTS), from `models.pipeline` (compute_domain_scores). Add argparse with `--cached-only` flag.

2. **Episode definitions**: Define OUT_OF_SAMPLE_EPISODES list of dicts following the research code example structure:
   - 1960s Urban Unrest: date range ("1965-01", "1968-12"), expected Elevated Tension (26-50), min_coverage 0.3
   - Watergate: date range ("1973-06", "1974-12"), expected Elevated Tension or low Crisis (40-60), min_coverage 0.3
   - Late 1980s Stability: date range ("1987-01", "1989-12"), expected Stable (0-25), min_coverage 0.5
   - 2016 Election: date range ("2016-11", "2017-01"), expected Elevated Tension (35-50), min_coverage 0.5
   - January 6 2021: date "2021-01", expected Crisis Territory (51-75), min_coverage 0.5

   Also define IN_SAMPLE_EPISODES from DEFAULT_ANCHORS with expected zones derived from their target scores (use score_to_zone on each target).

3. **`run_episode_backtest(calibrated_history: pd.Series, raw_scores: pd.Series) -> list[dict]`**:
   For each episode (in-sample + out-of-sample), extract the score from calibrated_history using _get_anchor_raw_score logic. For date ranges, average scores in the range. Compute the zone using score_to_zone. Compare to expected zone. Track: label, expected_zone, expected_range (if defined), actual_score, actual_zone, in_zone (bool), is_in_sample (bool), coverage_note (if no data found, mark as "NO_DATA" and note in results). Return list of episode result dicts.

4. **`run_loocv(raw_scores: pd.Series) -> list[dict]`**:
   For each of 5 DEFAULT_ANCHORS, remove it from the anchor list, call _fit_calibration with remaining 4, compute the predicted calibrated score for the held-out anchor using the new (a, b) coefficients and _get_anchor_raw_score. Compute deviation from target. Flag overfitting if deviation > 25 points. Return list of result dicts with: anchor label, target, predicted, deviation, overfitting flag, a, b coefficients.

5. **`check_ci_width(calibrated_history: pd.Series, unified_df: pd.DataFrame, calibration_coeffs: tuple, raw_df: pd.DataFrame = None) -> dict`**:
   Pick a crisis episode (2008) and a stability episode (mid-1990s) from the calibrated history. Compute bootstrap CI for each using compute_bootstrap_ci. Check whether the crisis CI lower bound is above the stability CI upper bound (non-overlapping = good discrimination). Return dict with crisis_ci, stability_ci, overlap status, gap size.

   IMPORTANT: The bootstrap CI function operates on the full unified_df (current time point). For historical CI comparison, instead extract the calibrated scores at the crisis and stability dates and note the point estimates. The CI width from the current computation indicates overall model uncertainty. Report the point-estimate gap between crisis and stability episodes alongside the current CI width to assess discriminability.

6. **`compute_overall_verdict(episode_results, loocv_results, anchor_residuals) -> dict`**:
   Apply the three CONTEXT.md pass/fail criteria:
   - Zone accuracy >= 75% (count episodes with data that land in correct zone / total with data)
   - Monotonic ordering: all crisis-labeled episodes score strictly higher than all stability-labeled episodes (hard gate). Only apply to episodes with data.
   - Calibration anchor residuals: no anchor > 15 points from target (fail), warn if > 10.
   Verdict is PASS only if all three criteria pass. Return dict with verdict, zone_accuracy percentage, monotonic_pass, residual_max, individual criterion results.

7. **`main()` function**:
   - Load data: read public/data/history.json to get the existing calibrated history (convert entries to pd.Series with DatetimeIndex). This avoids needing to re-run the full pipeline.
   - Also try to load the raw (uncalibrated) history by re-running the ensemble if --cached-only is not set, or by reading from a cached source. For simplicity in v1: read history.json entries as the calibrated history, and note that these scores are already calibrated.
   - Run episode backtesting: call run_episode_backtest
   - Run LOOCV: For LOOCV to work properly, we need raw (uncalibrated) scores. Since history.json contains calibrated scores, and _fit_calibration needs raw scores: the simplest approach is to load history.json as-is and note that LOOCV requires raw ensemble scores from a live pipeline run. If the live pipeline is not available (no FRED key), print a warning and skip LOOCV, or reverse-engineer raw from calibrated using current a,b coefficients as an approximation.

   PRACTICAL APPROACH: Since the existing history.json is the product of a demo/cached run and the full pipeline requires API keys and manual data, design validate.py to work in TWO modes:
   (a) **History-only mode** (default): Read history.json, run episode backtesting on calibrated scores. Skip LOOCV and CI (which require raw data). This mode works without any API keys.
   (b) **Full mode** (when --full flag is passed): Import and run the pipeline (fetch_all + compute_ensemble + calibrate), then run all validation tests including LOOCV and CI width. This requires FRED_API_KEY and cached data.

   Print console output: episode results table, pass/fail for each criterion, overall verdict.
   Store all results in a dict for later report generation (Plan 05-03).

8. **Console output format**: Print a clear summary table for episodes (label | expected zone | actual score | actual zone | pass/fail), LOOCV table (anchor | target | predicted | deviation | overfitting?), and overall verdict.

IMPORTANT: Do NOT use em dashes in any string literals or comments. Use commas, colons, or restructure instead.

Do NOT modify any existing production code files. validate.py is a standalone script that imports from existing modules.
