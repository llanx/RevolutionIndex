Add the following functions to models/validate.py (from Plan 05-01):

1. **`run_weight_sensitivity(calibrated_history: pd.Series) -> list[dict]`**:
   For each of the 5 models in MODEL_WEIGHTS:
   - Create two perturbations: +20% and -20% of that model's weight
   - Renormalize all weights to sum to 1.0
   - Since we work with calibrated history.json and cannot easily re-run the ensemble with different weights without the full pipeline, use a SIMPLIFIED approach:
     - Extract each model's approximate contribution using the weight ratios
     - Reweight the latest score proportionally: new_score = sum(model_score_i * new_weight_i) / sum(new_weights)
     - Since we do not have per-model scores from history.json alone, the full weight sensitivity requires the --full pipeline mode
   - For history-only mode: use a mathematical approximation. If current calibrated score is S and we perturb model M's weight by factor f, the maximum possible shift is bounded by |S * (f-1) * w_M / (1 + (f-1)*w_M)|. Report the theoretical maximum shift.
   - For --full mode: actually re-run _run_models_on_slice with patched MODEL_WEIGHTS and compute real shifts.

   PRACTICAL IMPLEMENTATION for history-only mode:
   - Read history.json, take the latest score as baseline
   - For each model, compute theoretical shift bounds based on weight perturbation math
   - Flag any perturbation where theoretical max shift > 25 points as potentially fragile
   - Print a note that exact sensitivity requires --full mode with live data

   For --full mode (when pipeline data is available):
   - Temporarily patch models.config.MODEL_WEIGHTS (save original, modify, compute, restore in try/finally)
   - Call _run_models_on_slice with the latest data slice using patched weights
   - Compute actual score shift from baseline
   - Flag if shift > 25 points (1 zone)
   - Return list of dicts: model, direction, baseline, perturbed_score, shift, fragile

2. **`check_inter_model_correlation(unified_df: pd.DataFrame = None, raw_df: pd.DataFrame = None) -> list[dict]`**:
   This requires running each model individually on the same data, which needs --full mode.
   - In --full mode: run each model function from MODEL_REGISTRY on the latest data slice independently, collect their individual scores
   - Compare all 10 pairwise combinations
   - Since we only have a single time point's worth of per-model scores (not a time series), we CANNOT compute Pearson correlation from a single observation. Instead, report the pairwise score differences: if two models produce very similar scores, they may be redundant, but note this is a weak test from a single time point.
   - For proper correlation, would need to run each model across all historical slices (computationally expensive). Flag this limitation in output.
   - In history-only mode: skip this test and print "Inter-model correlation requires --full mode"
   - Flag any pair where both scores are within 5 points as "similar" (note: this is not true correlation, just proximity at one time point)

3. **`check_spurious_trends(calibrated_history: pd.Series) -> dict`**:
   Perform the following automated checks on the calibrated history time series:
   a. **Monotonic increase check**: Check if scores are monotonically non-decreasing across ALL decades (1980s, 1990s, 2000s, 2010s, 2020s). A genuine stress indicator should have some decades with declines (e.g., 1990s). If ALL decades show increase, flag as potentially spurious.
   b. **Data boundary jump detection**: For each known data boundary year (1989, 1993, 2000, 2005, 2017, 2020), check if the score jumps by > 10 points within 2 years of the boundary. If so, flag the boundary year and jump magnitude.
   c. **Score saturation check**: Check if scores are at 0 or 100 for more than 3 consecutive data points. Saturation at the boundaries indicates normalization artifacts.
   d. **Decade-level summary**: Compute mean and range of scores per decade for quick visual inspection.

   Return dict with: monotonic_flag (bool), boundary_jumps (list of flagged years), saturation_periods (list), decade_summary (dict), overall_concern (str: "none", "minor", "major").

   This function works in history-only mode since it only needs the calibrated history time series.

4. **Update `main()` function** to call these three new functions and print their results:
   - Weight sensitivity: table with model | direction | baseline | shift | fragile?
   - Inter-model correlation: table with model_a | model_b | score_a | score_b | note
   - Spurious trends: checklist format with pass/flag for each check

   Store all results in the validation_results dict for the report generator (Plan 05-03).
