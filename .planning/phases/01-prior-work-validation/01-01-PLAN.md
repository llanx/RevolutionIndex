```markdown
# Model Assessment: [Model Name]

## 1. Theoretical Basis
- What the spec claims the model measures (cite spec section numbers)
- Source academic work (Turchin for PSI, Kahneman-Tversky for PLI, financial crisis literature for FSP)
- How faithfully the spec adapts the academic source

## 2. Component Review
For each component (MMP/EMP/SFD for PSI; 5 domains for PLI; FSSI/ETI for FSP):
- Spec definition: what the spec says this component captures
- Code implementation: what the code actually computes (cite file and line numbers)
- Simplifications: what was dropped from the full spec and whether that's defensible
- Known issues: what the critical reviews flagged, organized as:
  (a) Spec-level: problems in the theory/formula itself
  (b) Fix-level: whether the critical review's recommended fix is well-specified
  (c) Code-level: whether the fix was correctly implemented

## 3. Mathematical Fix Status
For each critical review item affecting this model:
- Issue ID and description
- Whether the fix was applied in code (cite file:line)
- Whether the application introduced new issues
- Current status: APPLIED-CORRECT / APPLIED-BUGGY / NOT-APPLIED / NOT-APPLICABLE

## 4. Verdict
Organize all findings into three categories:
- CONFIRMED: elements that hold up (critical review did not dispute AND code implements correctly)
- REVISED: elements changed from original spec with documented rationale
- FLAGGED: elements that are questionable and need attention in Phase 4 model-building

Include a 2-3 sentence summary suitable for the consolidated validation report.
```
