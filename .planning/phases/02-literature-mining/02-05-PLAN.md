**For each framework, write a full assessment including (locked decision):**
- **Theoretical Basis:** Core theory in 2-3 sentences
- **Required Inputs:** Specific variables the framework needs (list each)
- **Known Limitations:** Documented weaknesses, criticisms from the literature
- **Validation Track Record:** Where has it been tested? What were the results? (sample size, time period, accuracy)
- **Data Availability for US:** Can the required inputs be measured with available data? This is the PRIMARY assessment criterion (locked decision). Cross-reference against what was found in domain reviews and what is known about federal data APIs.
- **Assessment:** Overall viability for this project -- could this framework (or elements of it) supplement or replace the existing 3 models?

**Rules (locked decisions):**
- Assess each framework independently on its own merits, NOT compared against the existing 3 models. Comparison happens in a later phase.
- No artificial cap on number of frameworks -- include everything that passes the data-availability filter.
- The assessment is about "could we build this?" (data availability) not "should we build this?" (model selection is Phase 4).

**Known candidate datasets (from 02-RESEARCH.md, to be confirmed/expanded):**

1. **NAVCO 2.1** -- Nonviolent and Violent Campaigns and Outcomes, 389 campaigns, 1945-2013
2. **NAVCO 3.0** -- Event-level data, 100K+ events, 21 countries, 1991-2012
3. **Mass Mobilization Project** -- 162 countries, 1990-2020, 10K+ protest events
4. **UCDP** (Uppsala Conflict Data Program) -- Global conflict events, 25-death threshold

6. **V-Dem** -- 483 indicators, 202 countries, 1900-present

8. **OMG Dataset** (Dahl et al. 2025) -- 1789-2019 coverage
9. **Polity5** -- Regime characteristics, 1800-present
10. **Any additional datasets discovered in domain reviews**

**For each dataset, document:**
- **Full Name and Citation:** Author(s), year, official name
- **Coverage:** Countries, time period, event count or observation count
- **Key Variables:** What does it measure? What coding scheme?
- **Access Method:** URL, download process, API availability, license/cost
- **Update Status:** Last updated, update frequency, ongoing or completed
- **US Coverage:** Does it include US data? How many US observations? What types of events are coded for the US?
- **Applicability Assessment:** How could this dataset be used for US-focused model validation? Be honest about the zero-US-revolution-event constraint.

**Key insight from REQUIREMENTS.md:** The project explicitly excludes "ML prediction framing" because there are zero labeled revolution events for the US. This means these datasets serve different purposes:

- NAVCO: Campaign-level outcomes -- usable for threshold calibration (what level of mobilization has historically produced regime change?)
